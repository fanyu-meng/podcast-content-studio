# 播客 Show Notes 生成指南

> 📝 这个文档可以作为 Prompt 提供给任何 AI Chat，用于从播客转录稿生成高质量的 Show Notes。

---

## 🎯 任务说明

根据用户提供的播客转录稿（可以是文本、PDF、或任何格式的转录内容），生成一份高质量的 Show Notes，包括：
- 吸引人的标题
- 嘉宾介绍
- 主要内容描述
- 话题时间戳

**核心原则：具体 > 笼统，人味 > 光鲜，有趣 > 正确**

---

## 📋 工作流程

### Step 1: 读取并解析转录稿

读取用户提供的转录稿内容（文本、PDF、或其他格式），提取：
- 所有对话文本和发言者标识
- 转录稿中所有时间戳
- 单集编号（如有）

### Step 2: 识别嘉宾和关键话题

从对话内容中提取每位嘉宾的：
- 姓名（中文 + 英文）
- 当前角色 / 身份
- 对话中提到的背景和关键经历
- 显著成就或资历
- 提到的链接（网站、社交媒体）

同时识别 3-5 个候选亮点话题，这些应该是最具体、最令人惊讶或最引人注目的时刻。

### Step 3: 向用户提问（3 个问题）

在开始起草前，**一次性**向用户提出以下 3 个问题：

**问题 1：标题焦点**

展示从转录稿中提取的 3-5 个候选亮点话题，询问：
> "以下是我从转录中提取的几个最有亮点的细节，你希望标题围绕哪个点来写？也可以自己输入想强调的内容。"

选项：3-5 个候选话题 + "自定义输入"选项。

**问题 2：主要内容侧重点**

根据本期主题提供选项：
> "主要内容的侧重点你希望放在哪里？"

选项示例（根据实际内容调整）：
- 嘉宾的个人经历和故事
- 方法论和可操作的建议
- 行业观察和趋势分析
- 轻松有趣的对话和互动
- 自定义输入

**问题 3：风格偏好**

询问写作风格：
> "你希望这期 show notes 用什么风格来写？你也可以上传一个你喜欢的 show notes 示例作为参考。"

选项：
- 新闻联播戏仿体（正经语言写不正经内容，反差幽默）
- 故事悬念体（用"从...到..."叙事弧线，提问制造好奇）
- 直接吐槽体（主持人视角的主观评价和感受）
- 上传自己的示例作为参考
- 使用默认风格

如果用户上传了自定义示例，分析其风格特征（语气、结构、用词模式）来指导后续写作。

### Step 4: 起草标题

根据用户在 Step 3 选择的标题焦点，生成 3 个候选标题。

**标题检查清单：**
- 是否包含具体细节（数字、人名、地名）？
- 是否能让人停下来想点进去听？
- 是否避免了所有禁用词汇？

选择最佳的一个。格式：`EP{N}：{hook}`

**标题示例对比：**

| ❌ 坏（AI味） | ✅ 好（人味） |
|---|---|
| 探索脱口秀的魅力与挑战 | 从每个月40场开放麦到喜剧荒漠🇩🇪：我们在慕尼黑讲脱口秀 |
| 跨语言喜剧的深度对话 | 她用中英德三语讲脱口秀——逗笑德国人有这么难？ |
| AI时代的创业新思维 | 从清华程序员到硅谷solo创业：一个人怎么做AI公司？ |
| 深入探讨海外华人的职业发展 | 在德国年薪百万也想辞职？聊聊大厂程序员的中年危机 |

### Step 5: 撰写嘉宾介绍

**写作规则：**
1. 用**叙事体**而非简历体，读起来像在介绍一个朋友
2. 一句话概括核心身份，然后展开关键经历
3. 可以用"从A到B"的路径叙事串联身份转变
4. 收尾可以加一句有人情味的描述
5. 附上相关链接

**示例对比：**

❌ **简历体：**
> 张三，XX大学计算机科学博士，曾就职于Google、Meta，现任ABC公司CTO，专注于人工智能领域的技术创新与应用落地。

✅ **叙事体：**
> 张三，Google和Meta的前工程师，写了8年代码之后决定自己干。现在是ABC公司的CTO，带着一个5人小团队做AI工具。他说自己"终于不用写周报了"。

### Step 6: 撰写主要内容

**写作规则：**
1. 用**主持人第一人称**视角写，带有个人感受和评价
2. 内容要有**叙事弧线**，不是罗列话题
3. 加入主持人的**主观评价和感受**
4. 可以提到录制花絮、环境细节

**风格选项：**
- **新闻联播戏仿体**：正式语言写轻松内容，制造反差幽默
  - 例："展开了亲切会晤"、"气氛热烈祥和"
- **故事悬念体**：用"从...到..."叙事，以提问收尾
  - 例："从X到Y，TA经历了什么？"
- **直接吐槽体**：个人反应、幕后细节
  - 例："这期录完我嗓子都哑了"

**示例对比：**

❌ **AI摘要：**
> 本期节目邀请到了李刃，深入探讨了跨语言脱口秀的创作过程、海外华人喜剧的发展现状，以及如何在不同文化背景下建立个人品牌。

✅ **人味描述：**
> 李刃是全职脱口秀演员，也是一家俱乐部的老板。从创业失败到第一次登上脱口秀舞台，从几个月没人笑到获得欧洲各项脱口秀比赛冠军，李刃都经历了什么？这期可能是录制以来笑声最多的一次。如果我们的笑声过大过于刺耳，李刃全责。

### Step 7: 创建时间戳

#### Step 7a: 识别主要话题

按时间顺序识别**主要讨论话题**——不是话题内的每个子观点。

一个"主要话题"通常持续 3-8 分钟，围绕一个主题讨论后才转换到下一个。

**❌ 错误：拆分过细**
```
01:24 华人职场第一道坎：怎么开口跟老板聊升职？
02:33 核心观点：跟老板要建partner关系
03:46 "我很怕我的老板"：老李自曝早年觉得老板身上有光环
05:30 角色逆转：后来他升到了老板的同级
07:02 提供价值的时机：跟小时候考好了找家长要玩具是一个道理
```

**✅ 正确：合并为一条**
```
01:32 跟老板谈升职：从"我很怕我的老板"到变成他的上级——老李的partner关系论
```

#### Step 7b: 定位精确时间

- 时间**必须**来自转录稿中实际存在的 `发言人X MM:SS` 标记
- **绝不能**估算、插值或四舍五入
- 选择话题开始前最接近的发言时间标记

#### Step 7c: 撰写时间戳描述

格式：`MM:SS 标题关键词：具体描述内容`

**规则：**
- 用中文冒号 `：` 分隔标题和描述
- 每条时间戳单独一行，时间戳之间空一行
- 必须包含具体信息：人名、地名、数字、专有名词
- 可以用标点传达情绪：？！引号、破折号
- 输出必须是纯文本（无列表符号），可直接复制粘贴

**示例对比：**

| ❌ 坏（AI式摘要） | ✅ 好（人味迷你标题） |
|---|---|
| 19:00 讨论如何创建和运营一个喜剧俱乐部 | 19:00 创立"椒糖喜剧"：东东如何将运营经验转化为"经纪人"思维，慕尼黑中文开放麦为何场场爆满 |
| 31:41 分享高强度演出经历 | 31:41 "卷王"经历：盐盐在国内演出市场顶峰时期，一个月能讲40多场开放麦？！ |

#### Step 7d: 验证

回到转录稿验证**每一条**时间戳：
- 确认该时间点前后 30 秒内的文字确实对应描述的话题
- 如有错位，替换为正确的发言时间标记

#### 时间戳密度指南

- 30分钟以内：5-8 条
- 30-60分钟：8-12 条
- 超过60分钟：10-15 条
- 相邻时间戳最小间隔：2 分钟
- 超过60分钟的节目用 `H:MM:SS` 格式

### Step 8: 组装输出

使用以下模板输出最终 Show Notes：

---

## 📄 输出模板

```markdown
EP{N}：{标题}

**『本期嘉宾』**

{嘉宾1介绍}

{嘉宾链接（如有）}

**『主要内容』**

{主要内容描述}

**『时间戳』**

{MM:SS 标题关键词：具体描述}

{MM:SS 标题关键词：具体描述}

{MM:SS 标题关键词：具体描述}
...

**『播客渠道』**

小宇宙，苹果Podcast，Spotify

**『主播』**

{主播信息}

**『关于我们』**

{播客简介}

**『联系我们』**

{联系方式}
```

**格式规则：**
- 每个章节标题（`**『xxx』**`）单独一行
- 标题和内容之间空一行
- 每个章节之间空一行
- 每条时间戳后空一行
- 固定章节（播客渠道、主播、关于我们、联系我们）直接附加，不要修改

## 🚫 禁用词汇和表达

以下词汇/句式是典型 AI 生成痕迹，**绝对不要使用**：

### 标题禁用
- 深入探讨 / 深度解析 / 全面剖析
- 探索...的魅力 / 领略...的风采
- ...的N个关键 / ...的终极指南
- 干货满满 / 不容错过
- 赋能 / 破圈 / 硬核

### 正文禁用
- 在...的道路上 / 在...的浪潮中
- 让我们一起... / 带您走进...
- 不仅...更... / 既...又...（排比句式堆砌）
- 为听众带来了... / 为我们呈现了...
- 分享了宝贵的经验和见解
- 碰撞出思想的火花
- 令人深思 / 发人深省 / 引人入胜
- 满满的干货 / 收获颇丰

### 替代策略

用**具体事实**说话，而不是修饰性形容词：

| ❌ 坏 | ✅ 好 |
|---|---|
| 分享了精彩的创业经历 | 从0做到20亿，然后全部推翻重来 |
| 进行了深入的讨论 | 争了半小时，谁也没说服谁 |
| 展现了非凡的毅力 | 被拒了200次还在投简历 |

### 示例 2：嘉宾介绍（AI创业主题）

```markdown
**『本期嘉宾』**

Cathy Liu，连续创业者，也曾是"职场打工人"，做过公司的VP、COO。她从程序员开始一路走来，从清华走到硅谷。现为solopreneur.global的创始人，为一人公司或小团队服务的AI应用工厂和创业平台。帮助创业者缩短从想法到PMF(Product-Market Fit)验证的时间，加速MVP落地以及市场探索，快速得出go/no-go的结论。

Cathy做过从0到1的创业，也曾作为公司高管带领业务从0做到20亿人民币，她还曾以COO身份推动A轮公司扩张，实现规模化增长。

她是硅谷创业者，也是两个孩子的妈妈。

如果你对AI创业感兴趣，无论是想找点子、加速实现、验证市场或是寻找同路人，欢迎访问并注册： https://solopreneur.global
```

**手法分析：**
- 叙事体而非简历体，读起来像故事
- "从程序员开始一路走来，从清华走到硅谷"——用路径串联身份
- 最后一句"她是硅谷创业者，也是两个孩子的妈妈"——人味收尾
- 附带链接和 CTA

## 💡 双语处理

- 主体语言为中文
- 英文人名、品牌名、专业术语保留英文原文
- 不需要逐段翻译，但标题可以考虑中英双语版本
- 英文术语首次出现时可加括号注释，如 "PMF(Product-Market Fit)"

## 📝 使用说明
