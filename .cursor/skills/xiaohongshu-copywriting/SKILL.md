# 小红书播客宣传文案生成器

## Input

以下任一格式：

1. **PDF 转录稿** — 包含完整播客对话文字和时间戳的 PDF 文件
2. **Show notes Markdown** — 已生成的播客节目笔记（含标题、嘉宾、主要内容、时间戳等段落）

## Workflow

### Step 1: 读取并解析输入

**如果输入是 PDF 转录稿：**
- 提取所有对话文字和发言人标识
- 提取所有时间戳
- 识别期号（如有）
- 识别嘉宾姓名、身份、关键经历

**如果输入是 show notes Markdown：**
- 解析标题（EPxx：...）
- 解析『本期嘉宾』段落：嘉宾姓名、身份、背景
- 解析『主要内容』段落：核心叙事
- 解析『时间戳』段落：话题列表和对应时间

### Step 2: 提取"爆点素材"

从内容中识别 5–8 个最有传播潜力的素材点。按以下五类扫描，每类至少尝试找 1 个：

1. **反直觉观点** — "大家以为 X，其实 Y"；颠覆常识的判断
2. **具体数字/成就** — "从 0 到 20 亿"、"一个月 40 场"；可量化的冲击
3. **金句/名言** — 嘉宾原话中最有记忆点的句子，适合直接引用
4. **情感共鸣点** — 个人转折故事、失败经历、情绪爆发的瞬间
5. **实用方法论** — 可操作的建议、框架、清单；听众可以直接拿走用的

为每个素材点写一句话概括，格式：`[类型] 素材描述`

### Step 3: 交互 — 向用户提 3 个问题

使用 AskQuestion 工具一次性提出 3 个问题。

**Question 1: 文案类型**

> "你想生成哪种小红书文案？"

选项：
- 纯文字帖（一篇主推文案）
- 多角度文案（同一期生成 2–3 篇不同角度的帖子）
- 图文笔记文案（配合 carousel slides 的文字内容）
- 全部都要

**Question 2: 核心卖点**

展示 Step 2 中识别出的 5–8 个爆点素材作为选项，并问：

> "以下是我从内容中提取的传播爆点，你想围绕哪些来写？可多选 1–3 个。也可以自己输入想强调的内容。"

选项：Step 2 的素材列表 + "自定义输入"

**Question 3: 语气风格**

> "你希望用什么语气来写这篇小红书？"

选项：
- 闺蜜安利型（"姐妹们这期真的要听！"热情推荐，感性共鸣）
- 干货笔记型（"关于 XX 最清晰的一期"信息密集，理性有料）
- 故事悬念型（"他从 X 到 Y，中间经历了什么？"叙事弧线，引发好奇）
- 观点输出型（"听完这期我改变了一个想法"主观表达，引发讨论）
- 自定义（上传参考文案或描述想要的风格）

如果用户上传了自定义参考文案，读取并提取其风格特征（语气、结构、用词习惯）用于后续写作。

### Step 4: 生成标题（3 个候选）

根据用户选择的核心卖点和语气风格，生成 3 个候选标题。

**小红书标题规则（区别于 show notes 标题）：**
- 控制在 **20 字以内**（平台折叠限制，超出会被截断）
- 可用 emoji 开头增加视觉吸引力，但不超过 2 个
- **数字 + 反差 + 悬念** 三选二
- 不需要带 EP 编号（小红书用户不关心期号）

**标题模式参考：**
- `数字冲击型`："3 个方法让老板主动给你升职"
- `反转故事型`："从怕老板到变成老板的上级"
- `提问好奇型`："在德国年薪百万为什么还想辞职？"
- `金句提炼型`："「你的老板也是第一次当老板」"

**检查清单（每个候选标题必须过）：**
- [ ] 20 字以内？
- [ ] 包含具体信息（数字/人名/地名）？
- [ ] 让人想点进来看？
- [ ] 不在禁用词列表中？

展示 3 个候选标题让用户选择或修改。

Before finalizing, read [style-guide.md](style-guide.md) section "标题写作" for detailed rules.

### Step 5: 按类型生成正文

根据用户在 Step 3 选择的文案类型，生成对应格式的正文。应用用户选择的语气风格。

---

#### 类型 A: 纯文字帖

一篇完整的小红书帖子，可直接复制发布。

**结构：**

1. **Hook（前 2 行）** — 这是生死线。小红书展开前只显示前 2 行，必须在这里抓住注意力。用提问、反差、数字或金句开头。
2. **正文主体（3–5 个要点）** — 每个要点用 emoji 作 bullet，每段 1–3 句。围绕用户选择的核心卖点展开。
3. **金句引用（1–2 句）** — 用「」标记嘉宾原话，增加权威感和记忆点。
4. **个人感受** — 1–2 句主观推荐语，像朋友在安利。
5. **CTA + Hashtags** — 引导去听播客 + 话题标签。

**字数控制：** 正文 600–1000 字，不超过 1500 字。

Read [style-guide.md](style-guide.md) for Hook writing techniques and formatting rules.

---

#### 类型 B: 多角度文案

同一期节目生成 **2–3 篇**独立帖子，每篇围绕不同的爆点角度。

**用途：** 分多天发布，或 A/B 测试哪个角度更受欢迎。

**每篇的结构与类型 A 相同**，但：
- 每篇的标题和 Hook 完全不同
- 每篇聚焦不同的核心卖点
- 语气可以微调（如一篇偏干货、一篇偏故事）
- 篇与篇之间用分隔线 `---` 隔开

**标注格式：** 每篇开头标注角度，如 `【角度 1：职场方法论】`、`【角度 2：个人成长故事】`

---

#### 类型 C: 图文笔记文案

为 carousel 格式（多图笔记）提供每页 slide 的文字内容 + 帖子正文。

**结构：**

1. **封面 slide（第 1 页）**
   - 大标题（5–10 字，醒目有力）
   - 副标题（补充信息，1 行）
   - 角标/标签（如"播客笔记"、"职场干货"）

2. **内容 slides（第 2–6 页）**
   - 每页一个主题/要点
   - 页标题（加粗，5–10 字）
   - 要点内容（3–5 行，每行 1 句话）
   - 可配金句引用或数字高亮

3. **CTA slide（最后 1 页）**
   - 播客名称和收听引导
   - 平台信息
   - "关注获取更多"类引导

4. **帖子正文**
   - 配合图片的简短说明文字（200–400 字）
   - 包含关键信息概述 + CTA + hashtags

**总页数：** 5–8 页（含封面和 CTA 页）

### Step 6: 生成 Hashtags

为每篇文案生成话题标签，分两层：

**通用标签（3 个）** — 大流量池：
- 始终包含 `#播客推荐`
- 根据话题选择：`#职场成长` / `#海外生活` / `#创业` / `#科技` / `#个人成长` 等
- 根据受众选择：`#海外华人` / `#留学生` / `#职场人` 等

**内容标签（3–5 个）** — 精准匹配：
- 根据嘉宾身份定制（如 `#亚马逊` `#大厂经验`）
- 根据具体话题定制（如 `#升职加薪` `#向上管理`）
- 根据嘉宾/播客名定制（如 `#破壁圆桌` `#麦克老李`）

**格式：** hashtags 放在正文最末尾，每个标签之间用空格分隔。

### Step 7: 组装并输出

按用户选择的类型输出完整文案。

**输出格式：** `.md`（Markdown）

**输出文件命名：** `EP{N}_xiaohongshu_{type}.md`
- type = `text` / `multi` / `carousel` / `all`

**如果用户选择"全部都要"，按以下顺序输出，用一级标题分隔：**

```
# 纯文字帖

{文案内容}

---

# 多角度文案

{2-3 篇文案内容}

---

# 图文笔记文案

{slide 文案 + 帖子正文}
```

**最终检查清单（输出前逐条确认）：**
- [ ] 标题 20 字以内
- [ ] Hook 前 2 行有吸引力
- [ ] 无禁用词汇（参照 [style-guide.md](style-guide.md)）
- [ ] 正文字数在范围内
- [ ] Hashtags 已生成（通用 + 内容）
- [ ] CTA 已包含
- [ ] 可直接复制到小红书发布

## Anti-Patterns (NEVER use these)

**继承自 show notes skill 的禁用词：**

标题禁用：深入探讨, 深度解析, 全面剖析, 探索...的魅力, 干货满满, 不容错过, 赋能, 破圈

正文禁用：在...的道路上, 让我们一起, 带您走进, 分享了宝贵的经验和见解, 碰撞出思想的火花, 令人深思, 满满的干货, 收获颇丰

**小红书平台额外禁用：**

过度网感词：yyds, 绝绝子, 救命, 家人们, 集美们, 无语子（除非真的符合语境且用户要求）

营销号句式："看完这篇你就懂了", "建议收藏", "点赞 + 收藏 = 学会了", "早知道就好了", "后悔没早点看到"

标题党空壳："震惊！", "99% 的人不知道", "不看后悔", "太绝了"

**替代策略：** 用具体事实替代修饰词堆砌。
- 坏："分享了超有用的职场建议" → 好："3 个方法：第一次跟老板谈升职就成功了"
- 坏："嘉宾经历太励志了" → 好："从怕老板到变成老板的上级，用了 10 年"
- 坏："干货满满建议收藏！" → 好："我把他说的升职框架整理成了 3 步清单"
