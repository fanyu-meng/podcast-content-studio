# Show Notes 写作风格指南

## 核心原则

**具体 > 笼统，人味 > 光鲜，有趣 > 正确**

写 Show Notes 的目标是让人想点进来听，不是写一篇工作汇报。

---

## 标题写作

### 规则

1. 从转录文字中找到**最具体、最有冲击力**的一个细节作为 hook
2. 好标题 = 具体细节 + 情绪张力（反差/悬念/好奇）
3. 格式：`EP{N}：{hook}`
4. 可使用emoji（国旗🇩🇪、奖杯🏆等），但不滥用
5. 中英文混排自然即可，英文专有名词保留原文

### 标题生成流程

生成 3 个候选标题，按以下标准打分后选最佳：
- 是否包含**具体数字、人名、地名**？
- 读完是否会产生"这是什么？我想听"的反应？
- 是否避免了下方禁用词汇？

### 好标题 vs 坏标题

| 坏（AI味） | 好（人味） |
|---|---|
| 探索脱口秀的魅力与挑战 | 从每个月40场开放麦到喜剧荒漠🇩🇪：我们在慕尼黑讲脱口秀 |
| 跨语言喜剧的深度对话 | 她用中英德三语讲脱口秀——逗笑德国人有这么难？ |
| AI时代的创业新思维 | 从清华程序员到硅谷solo创业：一个人怎么做AI公司？ |
| 深入探讨海外华人的职业发展 | 在德国年薪百万也想辞职？聊聊大厂程序员的中年危机 |

## 嘉宾介绍写作

### 规则

1. 用**叙事体**而非简历体，读起来像在介绍一个朋友
2. 一句话概括核心身份，然后展开关键经历
3. 可以用"从A到B"的路径叙事串联身份转变
4. 收尾可以加一句有人情味的描述（如"也是两个孩子的妈妈"）
5. 附上相关链接（个人网站、社交媒体等）

### 示例对比

**坏（简历体）：**
> 张三，XX大学计算机科学博士，曾就职于Google、Meta，现任ABC公司CTO，专注于人工智能领域的技术创新与应用落地。

**好（叙事体）：**
> 张三，Google和Meta的前工程师，写了8年代码之后决定自己干。现在是ABC公司的CTO，带着一个5人小团队做AI工具。他说自己"终于不用写周报了"。

## 主要内容写作

### 规则

1. 用**主持人第一人称**视角写，带有个人感受和评价
2. 可以选择一种"人设"来写：
   - 新闻联播戏仿体（"展开了亲切会晤"、"气氛热烈祥和"）
   - 故事悬念体（"从X到Y，TA经历了什么？"）
   - 直接吐槽体（"这期录完我嗓子都哑了"）
3. 内容要有**叙事弧线**，不是罗列话题
4. 加入主持人的**主观评价和感受**
5. 可以提到录制花絮、环境细节（地点、天气、设备等）

### 示例对比

**坏（AI摘要）：**
> 本期节目邀请到了李刃，深入探讨了跨语言脱口秀的创作过程、海外华人喜剧的发展现状，以及如何在不同文化背景下建立个人品牌。

**好（人味描述）：**
> 李刃是全职脱口秀演员，也是一家俱乐部的老板。从创业失败到第一次登上脱口秀舞台，从几个月没人笑到获得欧洲各项脱口秀比赛冠军，李刃都经历了什么？这期可能是录制以来笑声最多的一次。如果我们的笑声过大过于刺耳，李刃全责。

## 时间戳写作

### 规则

1. 格式：`MM:SS 标题关键词：具体描述内容`，用中文冒号 `：` 分隔标题和描述；超过 60 分钟的节目用 `H:MM:SS`
2. 每条时间戳**单独一行**，时间戳之间**空一行**
3. 每条时间戳是一个**迷你标题**，不是内容摘要
4. 必须包含**具体信息**：人名、地名、数字、专有名词
5. 可以用标点传达情绪：？！引号、破折号
6. 短的可以很短（"讲脱口秀，赚到钱了"），长的可以有分号补充
7. 挑选节点的标准：**大话题的转换点**，不是话题内的每个子观点
8. 输出必须是**纯文本**，可直接复制粘贴到小宇宙/Apple Podcasts/Spotify
9. 粒度是**话题级别**：一个持续 3–8 分钟的完整讨论段落 = 一条时间戳。同一话题下的子观点、金句、例子不要拆成多条
10. 时间**必须**来自转录稿中实际存在的 `发言人X MM:SS` 标记，不能估算或推测
11. 写完后**必须**逐条回到转录稿验证：该时间点前后 30 秒内的文字是否确实是该话题的开始；如不对应，替换为正确的时间标记

### 示例对比

**坏（AI式摘要）：**
> 19:00 讨论如何创建和运营一个喜剧俱乐部

**好（人味迷你标题）：**
> 19:00 创立"椒糖喜剧"：东东如何将运营经验转化为"经纪人"思维，慕尼黑中文开放麦为何场场爆满

**坏（AI式摘要）：**
> 31:41 分享高强度演出经历

**好（人味迷你标题）：**
> 31:41 "卷王"经历：盐盐在国内演出市场顶峰时期，一个月能讲40多场开放麦？！

**坏（过于细碎——同一话题拆成多条）：**
> 01:24 华人职场第一道坎：怎么开口跟老板聊升职？
>
> 02:33 核心观点：跟老板要建partner关系
>
> 03:46 "我很怕我的老板"：老李自曝早年觉得老板身上有光环
>
> 05:30 角色逆转：后来他升到了老板的同级
>
> 07:02 提供价值的时机：跟小时候考好了找家长要玩具是一个道理

**好（合并为一条话题级时间戳）：**
> 01:32 跟老板谈升职：从"我很怕我的老板"到变成他的上级——老李的partner关系论

**坏（时间错位——转录稿中该内容实际在 24:32，偏差近 4 分钟）：**
> 28:15 巴甫洛夫的狗理论：你的终极目标是让老板看见你就分泌多巴胺

**好（使用转录稿中真实的发言时间标记）：**
> 24:32 巴甫洛夫的狗理论：你的终极目标是让老板看见你就分泌多巴胺

**好（参考截图风格）：**
> 02:10 从海牙精英到Etsy店主：我是如何把"不务正业"变成主业的？
>
> 23:40 如何面对"沉没成本"：如果神告诉你未来会很好，你敢不敢现在就离场？
>
> 53:54 商业奇招：订单太多做不完？我去eBay上雇了一群德国老太太帮我织毛衣！

## 禁用词汇和表达

以下词汇/句式是典型 AI 生成痕迹，**绝对不要使用**：

### 标题禁用
- 深入探讨 / 深度解析 / 全面剖析
- 探索...的魅力 / 领略...的风采
- ...的N个关键 / ...的终极指南
- 干货满满 / 不容错过
- 赋能 / 破圈 / 硬核

### 正文禁用
- 在...的道路上 / 在...的浪潮中
- 让我们一起... / 带您走进...
- 不仅...更... / 既...又...（排比句式堆砌）
- 为听众带来了... / 为我们呈现了...
- 分享了宝贵的经验和见解
- 碰撞出思想的火花
- 令人深思 / 发人深省 / 引人入胜
- 满满的干货 / 收获颇丰

### 替代策略

不要用修饰性形容词堆砌，而是用**具体事实**说话：
- 坏："分享了精彩的创业经历" → 好："从0做到20亿，然后全部推翻重来"
- 坏："进行了深入的讨论" → 好："争了半小时，谁也没说服谁"
- 坏："展现了非凡的毅力" → 好："被拒了200次还在投简历"

## 双语处理

- 主体语言为中文
- 英文人名、品牌名、专业术语保留英文原文
- 不需要逐段翻译，但标题可以考虑中英双语版本
- 英文术语首次出现时可加括号注释，如 "PMF(Product-Market Fit)"
